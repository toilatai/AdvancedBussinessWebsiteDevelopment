# Exercise 58: Fashion Website - Complete Implementation

This is a full-stack fashion website built with MongoDB, Express, and Angular, consisting of three main modules.

## Architecture Overview

### 1. **server-fashion** (Backend REST API - Port 4000)
- **Framework**: Express.js with MongoDB
- **Database**: FashionData (MongoDB)
- **Features**:
  - CRUD operations for Fashion items
  - Filter by Style
  - RESTful endpoints
  - CORS enabled

**Endpoints**:
- `GET /fashions` - Get all fashions (supports `?style=` query)
- `GET /fashions/:id` - Get fashion by ID
- `POST /fashions` - Create new fashion
- `PUT /fashions/:id` - Update fashion
- `DELETE /fashions/:id` - Delete fashion

---

### 2. **admin-fashion** (Admin Panel - Port 4001)
- **Framework**: Angular 18 (Standalone Components)
- **Features**:
  - View all Fashion items in a table
  - Add new Fashion
  - Edit existing Fashion
  - Delete Fashion (with confirmation)
  - WYSIWYG editor for details (using ngx-quill)
  - View Fashion details

**Routes**:
- `/fashions` - List all fashions
- `/fashions/add` - Add new fashion form
- `/fashions/edit/:id` - Edit fashion form
- `/fashions/view/:id` - View fashion details

---

### 3. **client-fashion** (Client/Public Website - Port 4002)
- **Framework**: Angular 18 (Standalone Components)
- **Features**:
  - Display fashions grouped by Style
  - Search by keyword
  - Filter by Style category
  - Modal view for fashion details
  - Responsive gallery layout

**Main Features**:
- Gallery grid display
- Search and filter functionality
- Modal popup for details
- Style-based grouping

---

## Setup Instructions

### Prerequisites
- Node.js v18+
- MongoDB running locally (default: `mongodb://localhost:27017`)
- Three terminal windows for running services

### Installation & Running

#### 1. Backend (server-fashion)
```bash
cd server-fashion
npm install

# Seed the database with sample data
npm run seed

# Start the API server
npm start
```
Server will run at **http://localhost:4000**

#### 2. Admin Panel (admin-fashion)
```bash
cd admin-fashion
npm install
npm start
```
Admin dashboard will run at **http://localhost:4001**

#### 3. Client Website (client-fashion)
```bash
cd client-fashion
npm install
npm start
```
Client website will run at **http://localhost:4002**

## Database Schema

### Fashion Collection
```javascript
{
  _id: ObjectId,           // Auto-generated by MongoDB
  title: String,           // e.g., "Bold Neon Ensemble"
  style: String,           // e.g., "Street Style", "Formal", "Casual"
  thumbnail: String,       // Image URL
  details: String,         // HTML content (WYSIWYG)
  createdAt: Date          // Auto-generated
}
```

## Troubleshooting

If MongoDB connection fails, ensure:
1. MongoDB is running: `mongod`
2. Database exists: `FashionData`
3. Connection string is correct in `server-fashion/index.js`

If API calls fail in Angular apps:
1. Verify `proxy.conf.json` is correct
2. Check that server-fashion is running on port 4000
3. Check browser console for CORS errors
